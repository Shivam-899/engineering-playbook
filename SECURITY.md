# Security Policy

> Version: 1.0.0
>
> Status: Stable
>
> Last Updated: 2026-06-30

---

# Supported Versions

The Engineering Playbook currently supports the latest version on the `main` branch.

Older versions may not receive security updates.

---

# Reporting a Vulnerability

Please do **not** report security vulnerabilities through public Issues.

Instead:

- Contact the repository maintainer privately.
- Include a clear description of the issue.
- Provide reproduction steps if possible.
- Include affected files or components.
- Suggest mitigations if known.

---

# Response Targets

We aim to:

- Acknowledge reports within 72 hours.
- Validate the report as quickly as practical.
- Keep the reporter informed throughout the process.
- Publish a fix before public disclosure whenever possible.

These targets are goals and may vary depending on the complexity of the issue.

---

# Responsible Disclosure

Please allow time for investigation and remediation before publicly disclosing a vulnerability.

Responsible disclosure helps protect users while fixes are prepared.

---

# Scope

This policy applies to:

- Source code
- Documentation
- Templates
- Playbooks
- Checklists
- GitHub workflows

---

# Out of Scope

The following are generally out of scope:

- Typographical errors
- Documentation wording issues
- Feature requests
- Unsupported versions

---

# Security Principles

This repository follows:

- Secure by Design
- Secure by Default
- Least Privilege
- Defense in Depth
- Continuous Improvement

---

# Thank You

Thank you for helping improve the security of the Engineering Playbook.

Responsible reporting benefits everyone.
