# Vision

Build a vendor-neutral engineering knowledge base containing practical standards, reusable templates, proven workflows, and AI-assisted development practices for building production-ready software.

This repository should remain useful regardless of programming language, framework, company, or AI coding assistant.

---

# Guiding Principles

- Quality over quantity
- Research before writing
- Technology-agnostic whenever possible
- Practical over theoretical
- Living documentation
- Continuous improvement

# Definition of Done

A document is considered complete when it:

- is research-backed
- contains practical guidance
- includes examples where appropriate
- is implementation-focused
- is reviewed before publishing
- follows the Engineering Constitution
